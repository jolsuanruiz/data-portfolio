Predictive Modeling

Using machine learning to predict customer churn.

Project Overview

This project focused on building a model to predict which customers are likely to cancel their subscription in the coming months. By anticipating churn, the business can proactively implement retention strategies and reduce revenue loss.

  • Purpose: Identify at‑risk customers and prioritize retention efforts.
  • Objective: Compare multiple classification algorithms and select the best performing model.
  • Tools: Python, Pandas, NumPy, Scikit‑Learn
  • Duration: 2 weeks

Approach

I began by exploring the telecom churn dataset, performing exploratory data analysis to understand the distribution of features and the churn rate. After cleaning and encoding categorical variables, I engineered features such as tenure groups and interaction terms. I then trained several classification models including Logistic Regression, Random Forest and Gradient Boosting. Hyperparameter tuning was performed using grid search and cross‑validation to select the best model.

Results & Insights

The Random Forest classifier achieved the highest accuracy and balanced precision/recall among the models tested. Feature importance analysis highlighted the impact of contract type, monthly charges and tenure on churn likelihood. The model allowed the business to target the top 20% of at‑risk customers with retention campaigns, reducing churn by an estimated 8%.

Reflection

This project underscored the value of feature engineering and model evaluation metrics beyond accuracy. I would extend the analysis by incorporating customer engagement data and building a real‑time prediction pipeline. Deploying the model via an API would also make it accessible to non‑technical stakeholders.
